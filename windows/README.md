## 🔧 API 設置

### 支持的 API 服務

| 服務 | 端點 | API Key |
|------|------|---------|
| **OpenAI 官方** | `https://api.openai.com/v1` | `sk-xxxx` |
| **Ollama (本地)** | `http://localhost:11434/v1` | `ollama` |
| **LM Studio** | `http://localhost:1234/v1` | `lm-studio` |
| **自定義** | `http://xxx.xx:xxxx/v1` | 按服務要求 |

### 設置步驟

```
1. 點擊右側 "⚙️ 設置" 面板

2. 填寫 API 設置：
   
   📝 API 端點:
   https://api.openai.com/v1
   
   🔑 API Key:
   sk-xxxxxxxxxxxxxxxxxxxx
   
   🤖 模型:
   gpt-4o-mini (推薦，速度快，成本低)
   
   ⏱️ 超時時間:
   30 (秒)
   
   🔄 重試次數:
   3

3. 點擊 "💾 保存配置"
   → 配置將保存在: C:\Users\xxx\.hermes\config\ai-repair-config.json

4. 後續啟動會自動加載配置
```

### 使用本地 AI (Ollama) - 完全免費

```bash
# 1. 下載並安裝 Ollama
# https://ollama.ai/download

# 2. 啟動 Ollama 服務
ollama serve

# 3. 在 AI File Repair 中設置:
# API 端點: http://localhost:11434/v1
# API Key: ollama
# 模型: deepseek-r1:1.5b

# 優點：
✓ 完全離線，無需網絡
✓ 無 API 成本
✓ 支持本地模型
```

## ❓ 故障排除

### 問題 1: 無法啟動程序

```
錯誤信息: "無法找到 Python"
或: "沒有模塊 named 'requests'"

解決方法:
1. 確保已安裝 Python
   下載: https://www.python.org/downloads/
   
2. 安裝依賴:
   pip install requests pyyaml pyinstaller
   
3. 或使用打包好的 .exe 文件 (推薦)
```

### 問題 2: API 連接失敗

```
錯誤信息: "API 請求失敗" 或 "Connection refused"

可能原因:
1. API 端點 URL 錯誤
2. API Key 無效或過期
3. 網絡連接問題
4. 服務器未運行 (本地 API)

排查步驟:
1. 檢查端點 URL 是否正確
   ✓ https://api.openai.com/v1
   
2. 檢查 API Key 是否有效
   ✓ 登錄 https://platform.openai.com 驗證
   
3. 測試網絡連接
   ✓ ping api.openai.com
   
4. 如果使用本地 API，確保服務已啟動
   ✓ Ollama: ollama serve
```

### 問題 3: 修復失敗

```
錯誤信息: "修復失敗" 或 "API 響應格式不正確"

可能原因:
1. 文件格式不支持
2. 文件過大
3. API 配額不足
4. AI 模型無法處理

解決方法:
1. 檢查文件類型是否在支持列表中
2. 將大文件分割成小塊 (建議 < 10MB)
3. 檢查 API 配額和用量
4. 尝試其他 AI 模型 (gpt-4o vs gpt-4o-mini)
```

### 問題 4: 修復結果不理想

```
問題: 修復後的文件仍有問題

解決方案:
1. 嘗試不同的修復類型
   例如: 自動檢測 → 代碼修復
   
2. 使用更強大的模型
   gpt-4o-mini → gpt-4o
   
3. 降低 temperature (更保守)
   0.3 → 0.1
   
4. 手動審核修復代碼
   預覽 → 確認 → 修復
```

### 問題 5: 程序閃退

```
可能原因:
1. Python 環境問題
2. 依賴版本不兼容
3. 系統兼容性問題

解決方法:
1. 使用打包好的 .exe 文件 (最穩定)
2. 以管理員權限運行
3. 更新 Windows 到最新版本
4. 安裝最新 Python 版本
```

## 🎯 最佳實踐

### 批量修復工作流程

```
1. 啟動程序
   ↓
2. 添加所有需要修復的文件
   ↓
3. 選擇合適的修復模式 (AI/標準)
   ↓
4. 選擇合適的修復類型
   ↓
5. (可選) 預覽第一個文件確認效果
   ↓
6. 點擊 "開始修復"
   ↓
7. 等待完成 (可做其他事情)
   ↓
8. 檢查修復結果
   ↓
9. 保存有用的修復配置
```

### 成本控制建議

```
使用 gpt-4o-mini (推薦):
• 速度快，成本低
• 適用於大部分修復任務
• 90%+ 的成功率

使用 Ollama (完全免費):
• 完全離線
• 無 API 成本
• 適用於敏感文件

避免使用 gpt-4o:
• 成本較高
• 除非處理非常複雜的修復
```

### 大文件處理

```
注意事項:
• 單個文件建議 < 10MB
• 大文件可能導致 API 超時
• 修復過程較長時請耐心等待

建議:
1. 將大文件分割成小塊
2. 分批修復
3. 優先修復重要部分
```
