## 🚀 Windows 版本使用指南

### 方法 1：直接運行 Python 腳本（推薦）

```bash
# 1. 安裝 Python (如果還沒有)
# 下載: https://www.python.org/downloads/
# 安裝時勾選 "Add Python to PATH"

# 2. 安裝依賴
pip install requests pyyaml

# 3. 運行程序
python gui_main.py
```

### 方法 2：打包成 Windows Executable (.exe)

#### 步驟 1：安裝 PyInstaller

```bash
pip install pyinstaller
```

#### 步驟 2：準備圖標（可選）

準備一個 256x256 的 PNG 或 ICO 文件，命名為 `icon.ico`

#### 步驟 3：打包

```bash
# 基本打包
pyinstaller --onefile --windowed --name "AI File Repair" --icon=icon.ico gui_main.py

# 帶圖標的完整打包
pyinstaller --onefile ^
  --windowed ^
  --name "AI File Repair" ^
  --icon=icon.ico ^
  --add-data "assets;assets" ^
  gui_main.py
```

#### 步驟 4：測試 .exe 文件

```bash
# 在 dist 目錄中找到生成的 exe 文件
# AI File Repair.exe

# 雙擊運行測試
```

## 📊 GUI 功能詳解

### 左側文件列表

| 按鈕 | 功能 | 快捷鍵 |
|------|------|--------|

## 🔧 故障排除

### 問題 1: 無法啟動

```
可能原因: 缺少 Python 或依賴

解決方法:
1. 檢查 Python 是否安裝: python --version
2. 安裝依賴: pip install requests pyyaml
3. 使用打包好的 exe 文件
```

### 問題 2: API 連接失敗

```
可能原因: API 端點或 Key 設置錯誤

解決方法:
1. 檢查端點 URL 是否正確
2. 檢查 API Key 是否有效
3. 測試 API 連接: python test_api.py
4. 如果使用本地 AI，確保服務已啟動
```

### 問題 3: 修復失敗

```
可能原因: 
- 文件格式不支持
- 文件過大
- API 限額不足

解決方法:
1. 檢查文件類型是否在支持列表中
2. 將大文件分割成小塊
3. 檢查 API 配額
4. 查看錯誤日誌了解具體原因
```

```batch
@echo off
chcp 65001 >nul

echo  AI File Repair - Windows打包腳本
echo ========================================

:: 檢查 Python

    echo [錯誤] 打包失敗

    exit /b 1

echo ========================================
echo  可執行文件位置: dist/AI File Repair.exe
echo.
echo  該程序可以在任何 Windows 電腦上直接運行
echo  無需安裝 Python 或任何依賴
echo.
pause
```

## 📋 最佳實踐

### 1. 批量修復

```
推薦工作流程:
1. 一次性添加所有需要修復的文件
2. 選擇合適的修復類型
3. 點擊 "開始修復"
4. 等待完成（可做其他事情）
5. 檢查修復結果
```

### 2. 大文件處理

```
注意事項:
• 文件超過 10MB 建議分批處理
• 修復過程可能需要較長時間
• 請確保有足夠的磁碟空間
• 修復後的文件會保存在原文件同目錄
```

### 3. API 成本控制

```
建議:
• 使用 gpt-4o-mini (成本低，速度快)
• 避免過大 max_tokens
• 本地 AI (Ollama) 完全免費
• 僅修復重要文件
```
